这个模型与之前看到的有错信道的停等协议一样吗？

显然没有什么区别
有区别，这个协议将思想抽象为了滑动窗口的思想，更利于协议的进一步发展与完善
有区别，名字不一样

在一位滑动窗口协议中，发送方和接收方的窗口大小均为1。发送方只能发送在窗口内的帧，接收方只接受在窗口内的帧。

在这个协议传输中，三号帧在传输过程中丢失，触发计时器超时，发送方重传该帧

七号帧的确认帧在传输过程中丢失，触发计时器超时，发送方重传该帧。

从下向上发的砖头代表“帧”，从上向下发的 √ 表示“确认帧”。

右侧是传输过程的伪代码

一位滑动窗口协议

创建

while(等待事件){

  if( 帧到达 ）

    if( 该帧为期望接收的帧 )

      接收该帧,提交给网络层,i++

      滑动接收窗口

    发送序号为窗口下限序号-1帧的确认帧

do{

  发送第 i 帧

  为每个帧设定计时器

  if（ 收到期望的确认帧 && 未超时 )

    滑动发送窗口，i++

  清空计时器

}while ( 还需继续发送帧 )
